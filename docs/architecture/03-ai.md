# Architecture: AI

> Status: Draft (Phase 2) · Updated: 2026-07-05 · See [[langgraph-agents]], [[mcp]].

## Purpose
How Hermes uses LLMs safely and deterministically: models, the tool contract, validation boundaries,
and observability. Orchestration mechanics are in [10-agent.md](./10-agent.md).

## Model policy
- Default to the latest Claude models (Opus 4.8 `claude-opus-4-8`, Sonnet 5 `claude-sonnet-5`, Haiku 4.5
  `claude-haiku-4-5-20251001`). Centralize model IDs in one config (no scattering).
- **Tiered routing:** cheap/fast model (Haiku) for classification/routing; stronger (Sonnet/Opus) for
  negotiation reasoning and plan synthesis. Escalate only when needed.

## Tool contract (the safety boundary)
- Every capability the model can invoke is a **registered tool** with a **Zod/JSON-Schema** input schema
  and an **authorization/policy check**. No free-form shell, network, or SQL.
- LLM output is **untrusted input**: validate against schema before acting; reject/repair on mismatch.
- Money tools (`pay`, `settle`) require the **policy gate** to pass first (see [20-payment-flow.md](./20-payment-flow.md)).
- Tools are the same whether called by our graph or an external agent via MCP (one definition, in
  `packages/types` schemas + `packages/shared`/`services` impls).

## Determinism & guardrails
- Bound every loop (max negotiation rounds, max tool retries) → guaranteed termination.
- Structured outputs (tool calls / JSON schema) over free text for anything acted upon.
- Idempotent tools where they touch state; pass an idempotency key for money tools (the nonce).

## Observability
- Each run: `run_id`, node transitions, tool calls (+args hashes), token usage, cost, latency.
- Spend trace: every policy-gate decision + Signer signature + settlement linked to `run_id` and Order.
- Traces persisted (Supabase) and surfaced in the web app (agent console + workflow canvas).

## Prompting
- System prompts encode Hermes protocol + guardrails; few-shots for negotiation patterns as MCP
  **prompts** where useful. Keep prompts versioned in-repo.

## Open questions

- Eval harness for negotiation quality + guardrail adherence (add in testing phase).
