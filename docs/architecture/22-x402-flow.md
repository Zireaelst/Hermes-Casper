## Purpose

protocol layer; [20-payment-flow.md](./20-payment-flow.md) wraps it with policy/orders/receipts.

## Roles
- **Resource server** â€” an x402-protected Hermes (or third-party) service endpoint.
- **Client** â€” the buying agent's x402 client (in `services/agent-runtime` via `packages/shared` adapter).
- **Signer** â€” holds the buyer's operational key, produces the EIP-712 authorization (ADR-003).
- **Facilitator** â€” Hermes-hosted; `/verify` + `/settle`, holds the funded gas key (ADR-005).

## Wire details (verbatim from research)
- **Headers:** `PAYMENT-REQUIRED` (402), `PAYMENT-SIGNATURE` (retry), `PAYMENT-RESPONSE` (200). All
  base64-encoded JSON. Optional facilitator `Authorization` via `FACILITATOR_API_KEY`.
- **PaymentRequirements:** `scheme:"exact"`, `network` (CAIP-2 `casper:casper-test`), `payTo` (66-hex
  account hash), `amount` (base-unit string), `asset` (64-hex CEP-18 package hash), `extra.{name,version,
  decimals}`, `maxTimeoutSeconds`.
- **authorization:** `from`, `to`, `value`, `validAfter`, `validBefore`, `nonce(32 bytes)`; signature 65
  bytes; `publicKey` algo-prefixed.
- **Facilitator validation:** `to==payTo`, `value==amount` (non-zero), asset well-formed,
  `validAfter<=now<=validBefore`, `validBefore-now>=6s`, name/version present, signature verifies.

## Idempotency & failure handling
- **Nonce = idempotency key.** Persist each nonce; reject reuse. Retries of `/settle` for the same nonce
  must not double-spend â€” track (nonce â†’ deploy hash â†’ status).
- Pre-authorize with **`/verify`** (cheap, no chain write) during negotiation/quote; only `/settle` at
  Order execution.
- Deploy finality is async â†’ Payment states: `quoted â†’ authorized â†’ verifying â†’ settling â†’ settled |
  failed | expired`. On `failed`/timeout, the authorization expires (validBefore) and is safely retryable
  with a fresh nonce.
- **Fail closed:** any ambiguity (unknown deploy status, facilitator error) holds the Order in `settling`
  and alerts â€” never optimistically mark paid.
